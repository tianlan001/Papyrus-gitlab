        Requirements

        - DnD of usesecase into a subject of a usecase diagram (id=org.eclipse.papyrus.uml.diagram.usecase.Req_010): 
         allows to drag and drop a usecase from model explorer into a subject of a usecase diagram
        - DnD of usecase from root without any predefined subject into a target subject (id=org.eclipse.papyrus.uml.diagram.usecase.Req_010.1): 
         allows to drag and drop a usecase with an empty subject from the root model element into a target subject of a usecase diagram.
        - DnD of usecase from root with an identical subject into a target subject (id=org.eclipse.papyrus.uml.diagram.usecase.Req_010.2): 
         allows to drag and drop a usecase with an identical subject from the root model element into a target subject of a usecase diagram.
        - DnD of usecase from root with a different subject into a target subject (id=org.eclipse.papyrus.uml.diagram.usecase.Req_010.3): 
         allows to drag and drop a usecase with a different subject from the root model element into a target subject of a usecase diagram.
        - Propose to keep or switch the subject and/or the owner (id=org.eclipse.papyrus.uml.diagram.usecase.Req_030): 
         The DnD action should open a dialog box to allow user to select if it keep or switch the subject and/or the owner

        DnD System

        DnD of usecase from root without any predefined subject into a target subject

          - traces to DnD of usecase from root without any predefined subject into a target subject
        A user can drag and drop a source usecase from model explorer into a target subject in a usecase diagram where 
        - the source usecase belongs to the the root model element, that is a different owner from the target subject
        - the subject of the source usecase is empty

        DnD of usecase from root with a different subject into a target subject

          - traces to DnD of usecase from root with an identical subject into a target subject
        A user can drag and drop a source usecase from model explorer into a target subject in a usecase diagram where 
        - the source usecase belongs to the the root model element, that is a different owner from the target subject
        - the subject of the source usecase is different from the one of the target subject

        DnD of usecase from root with an identical subject into a target subject

          - traces to DnD of usecase from root with a different subject into a target subject
        A user can drag and drop a source usecase from model explorer into a target subject in a usecase diagram where 
        - the source usecase belongs to the the root model element, that is a different owner from the target subject
        - the subject of the source usecase is identical to the target subject
